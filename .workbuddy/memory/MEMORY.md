## 项目：立杰人力资源管理系统

- **项目路径**：`e:\PostGraduate\ProgrammingStudy\AutoScanMind\Li---Jie\li_jie_hr\`
- **技术栈**：Python 3 + pywebview 5.x + HTML/CSS/JS（纯本地 Chromium 窗口，无网络依赖）
- **运行**：`pip install webview>=5.0.0 && python main.py`，或双击 `run.bat`
- **打包**：`pyinstaller build.spec`（待添加）
- **状态**：2026-04-11 完成 pywebview + FastAPI v2 版本全部功能，语法检查通过，数据验证通过
- **数据库**：新增 orders 表（含 year/month/remark）和 order_models 关联表，work_records 加 order_id FK，支持 v1→v2 迁移
- **已知 bug 修复**：型号单价表 API 返回的 prices 字典 key 从 tuple 改为字符串 `"model_id,sub_dept_id"`，避免 FastAPI jsonable_encoder 的 unhashable list 错误
- **种子数据**：`seed_data.py` 生成 14 名员工、6 个型号、3 个订单（ORD-001~003）、97 条做货记录（2026-04）
- **用户需求（v2）**：资源总览（首页导航）+ 订单管理（CRUD+型号联动）+ 成员/部门/型号单价/做货编辑/总工资表/快捷计算，共 8 个功能页
- **做货编辑**：选订单 → 选型号 → 添加行，订单/型号列为下拉框（联动），Tab 在员工格之间跳转；新增"切换工资视角"按钮（只读显示每个格子的对数×单价）；保存用 order_id
- **快捷计算**：上方主表格（型号行×员工列，可选型号+可删除行，填对数实时算工资，与做货编辑一致的工资视角切换和Tab跳转）；下方"单价编辑"= 内嵌型号单价表（型号为文本输入可自由填入，autoSave自动落库；有删除按钮但至少保留一行）；`pricesByName` = `{"modelName,subDeptId": price}`；主表格新增行始终选单价编辑第一个型号；主表格可完全删空后可点"+"添加行；员工列顺序与成员管理对齐；模板保存/加载/删除（POST/DELETE /api/price-templates）；工具栏有 **💾 保存**按钮（手动保存主表格结构+对数+单价编辑行，保存后按钮变绿显示"✓ 已保存"）；autoSave（500ms防抖，填对数/改单价后自动触发）+ **一键清空**按钮；进入页面自动加载上次年月保存的状态
- **成员管理**：紧凑卡片（姓名/性别♀♂/大部门/小部门 + 详情/编辑/增扣/删除按钮）；点击「详情」跳转到成员详情页
- **成员详情**：GET /api/employees/{id}/work-history，返回员工入职以来所有月份的做货记录（按年月倒序）；顶部汇总卡片显示累计做货对数/做货工资/人工增扣/总收入；每月一个折叠块，含订单号/型号/对数/单价/工资明细，人工增扣行高亮显示
- **配色规范**：成员姓名=蓝色(#2563eb)，大部门=橙色(#d97706)，小部门=绿色(#059669)；所有页面统一应用
- **表格自适应显示**：做货编辑和快捷计算的员工列（>8人时）自动拆分为多个独立表格纵向排列，每表含完整固定列（订单号/型号/行合计/操作）；大额工资使用 fmtCompact 紧凑格式（<1万显示¥1234.56，≥1万显示¥1.2万，≥100万显示¥100.0万），鼠标悬停显示完整数值；紧凑模式下自动缩小字号（11px/10px）；分组标签（"第X组/共Y组"）分隔各表格
- **系统设置页面**：左侧导航（外观/字体/表格/颜色主题/布局/数据），支持深色模式、圆角/阴影风格、字体大小（基础/标题/大标题）、字体族、表格字号/行高/斑马纹/紧凑模式/分组数量、5种颜色预设（蓝色/绿色/紫色/橙色/红色）+自定义颜色、侧边栏宽度/内容边距/卡片间距、导出/导入/重置设置；设置存储在 localStorage，自动应用
